# CLAUDE.md

## Commands

```bash
pnpm install          # pnpm 11 / Node >= 22.13
pnpm run dev          # dev server on port 4321 (strictPort, auto-opens)
pnpm run build        # tsc --noEmit + vite build ŌåÆ dist/
pnpm run typecheck    # tsc --noEmit only
pnpm run generate:api # openapi-typescript swagger-v3.json -o src/types/api.d.ts
```

There is no test suite and no `lint` script (an ESLint 9 flat config exists; run `npx eslint src` manually if needed). `pnpm run build` is the only verification gate and also runs inside the Docker build on deploy.

## Architecture

React 19 + Vite 6 (SWC) + TailwindCSS 4 + shadcn/ui + React Router 7 (data mode). UI text and most comments are in Spanish.

**Entry flow:** `src/main.jsx` ŌåÆ `src/App.jsx` (`ThemeProvider` ŌåÆ `RouterProvider`) ŌåÆ `src/routes/routes.jsx` (root router).

### Feature-based structure

`src/features/README.md` is the authoritative convention doc. Summary:

- `src/shared/` ŌĆö cross-cutting layer: `ui/` (shadcn components), `api/axiosService.js` (single Axios instance with JWT auto-refresh interceptors, base URL from `VITE_API_URL`), `auth/` (store, guards, RBAC), `layouts/`, `lib/`, `pages/`, `theme/`. No business logic.
- `src/features/<feature>/` ŌĆö business domains (`fichas`, `inventario`, `tickets`) with standard segments (`api/`, `components/`, `hooks/`, `store/`, `pages/`, `schemas/`, `constants/`, `types/`, `routes.tsx`, `index.ts`). Only create segments a feature actually needs.
- **Dependency rule:** `shared` never imports from `features`. Features import other features only through their public `index.ts` (e.g. `@/features/inventario`), never internal files.
- Each feature exposes its routes via `routes.tsx`; the root router just spreads them (`...fichasRoutes`).

### Incremental TypeScript

JS and TS coexist deliberately (`allowJs: true`, `checkJs: false` ŌĆö only `.ts/.tsx/.d.ts` are type-checked). Rules from `src/plan/migracion-incremental-typescript.md`:

- **New features are written entirely in `.ts/.tsx`.** Legacy code and `shared/` stay `.js/.jsx`; their boundary is typed with co-located `.d.ts` files (e.g. `shared/api/axiosService.d.ts`).
- `verbatimModuleSyntax` is on: use `import type` / `export type` for type-only imports.
- Use `@ts-expect-error`, never `@ts-ignore`.
- React 19 conventions: no `React.FC` (type props with your own `interface`/`type`); `ref` as a normal prop (no `forwardRef`); `import type { JSX } from "react"` instead of the deprecated global `JSX`.
- `src/types/api.d.ts` is auto-generated by `pnpm run generate:api` (`openapi-typescript`) from the committed `swagger-v3.json` ŌĆö don't hand-edit it. Regenerate `swagger-v3.json` from `swagger.json` (OpenAPI 2.0, from the backend) via `npx swagger2openapi swagger.json -o swagger-v3.json` when the API changes.

### Auth & RBAC

- **Route protection is loader-based** (React Router 7 data mode), not component-based: `requireAuth` gates the `MainLayout` route; `protect(PERMISSIONS.X)` is the per-route loader factory. Both live in `src/shared/auth/guards.js` and redirect before render. This is UI gating only ŌĆö the backend re-validates.
- Three user groups (Administrador, Tecnico, Administrativo) mapped to permission strings in `src/shared/auth/permissions.js`; checks go through `can()` in `src/shared/auth/authz.js`.
- Auth state: Zustand store `src/shared/auth/authStore.js`, persisted to sessionStorage. Feature-local state lives in each feature's `store/` segment.

### Path alias

`@/` ŌåÆ `src/` (configured in both `vite.config.js` and `tsconfig.json`).

## Migration context

The move from the old `src/components`-based layout was executed per the plans in `plans/` (read them before restructuring anything). Intentional quirks preserved from the migration: legacy filenames keep their typos (e.g. `InventorySerch.jsx`); folder names correct them (`inventario`).

## Gotchas

- `.env` is committed with real values (API URLs). Do not add secrets to it.
- `pnpm-lock.yaml` is committed and required for Docker builds; `package-lock.json` is gitignored.
- ESLint settings pin React `18.3` while the project uses React 19 ŌĆö known mismatch, don't "fix" it.
- Deploy CI (`.github/workflows/deploy.yml`) triggers on push to `main` (SSH + docker compose on the server). Day-to-day work happens on `development`.
- Commits follow Conventional Commits (`feat(scope): ŌĆ”`, `fix(scope): ŌĆ”`, `chore(scope): ŌĆ”`).
