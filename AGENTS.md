# AGENTS.md

## Quick Start

```bash
pnpm install          # pnpm 11 / Node >= 22.13
pnpm run dev          # dev server on port 4321 (auto-opens, strictPort)
pnpm run build        # tsc --noEmit + vite build → dist/
pnpm run typecheck    # tsc --noEmit only
pnpm run generate:api # openapi-typescript swagger-v3.json -o src/types/api.d.ts
```

**No test suite or lint script exists.** ESLint 9 flat config is present (`eslint.config.js`, covers JS and TS) but `package.json` has no `lint` script. `pnpm run build` is the only verification gate (also runs inside the Docker build on deploy).

## Architecture

React 19 + Vite 6 (SWC) + TailwindCSS 4 + shadcn/ui + React Router 7 (data mode). JS/TS hybrid (see below).

**Entry flow:** `src/main.jsx` → `src/App.jsx` (`ThemeProvider` → `RouterProvider`) → `src/routes/routes.jsx` (root router).

**Feature-based structure** (authoritative doc: `src/features/README.md`):
- `src/shared/` — cross-cutting layer, no business logic: `ui/` (shadcn), `api/axiosService.js` (single Axios instance, JWT auto-refresh interceptors, base URL from `VITE_API_URL`), `auth/`, `layouts/`, `lib/`, `pages/`, `theme/`.
- `src/features/<feature>/` — business domains (`fichas`, `inventario`, `tickets`) with standard segments (`api/`, `components/`, `hooks/`, `store/`, `pages/`, `schemas/`, `constants/`, `types/`, `routes.tsx`, `index.ts`). Only create segments a feature needs.
- **Dependency rule:** `shared` never imports from `features`. Cross-feature imports only through the feature's public `index.ts` (`@/features/inventario`), never internal files.
- Each feature exposes its routes via `routes.tsx`; the root router spreads them (`...fichasRoutes`).

**State:** Zustand. Auth store in `src/shared/auth/authStore.js` (persisted to sessionStorage); feature-local stores in each feature's `store/` segment. Theme via React Context (`src/shared/theme/ThemeContext.jsx`).

**Auth & RBAC:** Route protection is **loader-based**, not component-based: `requireAuth` gates the `MainLayout` route, `protect(PERMISSIONS.X)` is the per-route loader factory — both in `src/shared/auth/guards.js`, redirecting before render. Three user groups (Administrador, Tecnico, Administrativo) map to permission strings in `src/shared/auth/permissions.js`; checks go through `can()` in `src/shared/auth/authz.js`. UI gating only — the backend re-validates.

## TypeScript (incremental)

JS and TS coexist deliberately: `allowJs: true`, `checkJs: false` — only `.ts/.tsx/.d.ts` are type-checked. Rules (from `src/plan/migracion-incremental-typescript.md`):

- **New features are written entirely in `.ts/.tsx`.** Legacy code and `shared/` stay `.js/.jsx`; their TS boundary is typed with co-located `.d.ts` files (e.g. `shared/api/axiosService.d.ts`).
- `verbatimModuleSyntax` is on: use `import type` / `export type` for type-only imports.
- Use `@ts-expect-error`, never `@ts-ignore`.
- React 19: no `React.FC`; `ref` as a normal prop (no `forwardRef`); `import type { JSX } from "react"`, not the deprecated global `JSX`.
- `src/types/api.d.ts` is auto-generated by `pnpm run generate:api` (`openapi-typescript`) from the committed `swagger-v3.json` — don't hand-edit. Regenerate `swagger-v3.json` from `swagger.json` (OpenAPI 2.0, from the backend) via `npx swagger2openapi swagger.json -o swagger-v3.json` when the API changes.

## Conventions

- **Path alias:** `@/` → `src/` (`vite.config.js` + `tsconfig.json` — `jsconfig.json` was replaced by `tsconfig.json`)
- **Component files:** PascalCase `.jsx`/`.tsx` (e.g., `FichaTecnicaForm.jsx`); utilities/hooks camelCase `.js`/`.ts`
- **UI components:** shadcn/ui in `src/shared/ui/` (new-york style, Radix primitives, Lucide icons)
- **Language:** UI text and most code comments are Spanish
- **CSS:** TailwindCSS 4 with `tailwindcss-animate`; theme via class strategy (`light`/`dark` on `<html>`)
- Legacy filenames keep their typos (e.g. `InventorySerch.jsx`) — folder names correct them (`inventario`). Don't rename.

## Key Gotchas

- `.env` is committed with real values (API URLs). Do not add secrets.
- `package-lock.json` is gitignored. `pnpm-lock.yaml` is committed (required for Docker builds).
- ESLint config targets React 18.3 in settings but project uses React 19 — known mismatch, don't "fix" it.
- Dockerfile uses Node 22 (bookworm-slim) for build (pnpm 11 requires Node >= 22.13).
- Deploy CI (`.github/workflows/deploy.yml`) triggers on push to `main` via SSH + docker compose.
- The migration from the old `src/components` layout was executed per the plans in `src/plan/` — read them before restructuring anything.
